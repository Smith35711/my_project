## 状态定义

| 状态 | 含义 |
|------|------|
| 📝 待评审 | 架构师已记录，待与客户确认 |
| ✅ 已确认 | 需求确认完毕，待设计 |
| 🎨 设计中 | 架构师正在编写设计文档 |
| 🔨 开发中 | 开发工程师正在实现 |
| 🧪 测试中 | 测试工程师正在验证 |
| ✔️ 已完成 | 已通过测试，合并到主分支 |
| ❌ 已取消 | 需求被废弃 |

## 优先级定义

| 级别 | 含义 |
|------|------|
| P0 | 核心功能，没有则系统不可用 |
| P1 | 重要功能，第一版必须交付 |
| P2 | 一般功能，可延后交付 |
| P3 | 锦上添花，后续版本考虑 |
